Administrative tables help page

Administrative tables page allows to access all configuration tables of the system. All users who are given access to this page also have full access rights to all the data shown here.

Tables accessible from here can be categorised in three groups:

Access right management. Table Users contain information about all the users who can log in the system.

- Fields Login name and User password contain the information for logging into the system and should be nonempty (note that passwords are stored in database in unencrypted form, i.e. they are available to everybody having access to Users table). 
- Field User rights contains the information about the users access rights and should contain one of the following values: A0, A1,  G0, G1,  O1, O1, V0, V1. Prefixes "V", "O", "G" and "A" correspondingly denote the following access rights:
        V -  view only access. All tables can be viewed, no changes are allowed.
        O - user data access. All tables can be viewed and new person entries added. Editing and deleting is possible only for data entered by this user. Also new samples/aliquots can be created only to persons/samples entered by this user.
        G - group data access. All tables can be viewed and new person entries added. Editing and deleting is possible only for data entered by user from the same MolPAGE partner. Also new samples/aliquots can be created only to persons/samples entered by user from the same MolPAGE partner.
        A - full access. All tables can be viewed and all data added/deleted/edited.
Suffix "0" denotes that the user doesn't have the access to Administrative tables page, suffix "1" denotes that the user has such access. Note, that since passwords are not encrypted, the access rights values V1, O1 and G1 should be avoided.
- Field Partner may contain a foreign key to Partner, this user is associated with, however this may be left undefined.

User Id's are used as foreign keys in practically all other tables, therefore it will be possible to delete a user from the system only if the database doesn't contain any records created or modified by this user.


Configuration of web pages. This group contains three tables: Person matadata, Sample metadata and Aliquot metadata. Each of them allows to copnfigure fields that are respectively shown in Persons, Samples and Aliquots web pages. For each of the properties available through these web pages the following options can be configured:

- Short name. The column name shown on List of persons/sample/aliquots  page.
- Long name. The column name shown on Edit  page.
- View column number. Column position on List of persons/sample/aliquots  page. Value "0" means that the property is not shown. Column with a larger View column number is displayed to the right from column with smaller (non-zero) number, however these numbers are not required to be consecutive.
- Sort in view. If true data on List of persons/sample/aliquots  page will be sorted according to the values of this property on the click on column name.
- Report column number. Column position on Reports page.  (See for View column number more details.)
- Show in report. If true the property will be shown in Reports page by default.
- Sort in report. If true data on Reports page will be sorted according to the values of this property on the click on column name.
- Filter in report. If true the property will be available in search filter on Reports page.

Configuration of fixed vocabularies . This group contains 18 tables: Aliquot concentration units, Aliquot volume units, Aliquot fitnesses, Aliquot locations, Person disease stata, Person ethnicities, Planned users, Person sources, Processing options, Processing protocols, Reception stata, Relation types, Sample locations, Sample types, Sample subtypes, Person genders, Storage conditions, Transport conditions.

- Most of these tables (apart from Relation types, Sample types and Sample subtypes)  contain just fixed vocabulary terms for specific property and can be freely edited (however, you will be able to delete an entry only if there will be no records in the database referencing it).
-  Table Sample types in addition contains field Abbreviation, used for automatic generation of sample id's. Also the value of this field may be freely changed (however change of the value will not affect id's of samples that are already in database).
- Table Sample subtypes contains list of all subtypes for a given sample type. To add a new sample subtype you already need to have a corresponding sample type in Sample types table.
- Table Relation types lists all relations that can be defined between persons in Persons page.  The contents can be freely edited, however, since
reverse relations are identified just by names in this table you should exersise care in doing this.
